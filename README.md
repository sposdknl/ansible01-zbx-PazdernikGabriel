# Ansible Zabbix Agent 2 - Instalace a konfigurace

Nejprve jsem nakonfiguroval druhou síťovku ve virtuální appliance, aby byla správně připojená k laboratorní síti, a udělal jsem screenshot tohoto nastavení pro dokumentaci. Poté jsem naklonoval repozitář z GitHubu příkazem `git clone https://github.com/sposdknl/ansible01-zbx-PazdernikGabriel.git` a přešel do složky `cd ansible01-zbx-PazdernikGabriel`. Nainstaloval jsem potřebnou Ansible kolekci pro Zabbix příkazem `ansible-galaxy collection install community.zabbix`. Hlavním úkolem bylo opravit playbook `install-zabbix-agent2.yml`, který měl chybnou syntaxi - původně chyběly sekce `hosts`, `become` a `tasks`. Po opravě playbook správně definoval instalaci Zabbix Agent 2 ve verzi 7.0 s hostname "bastion", listen IP "{{ ansible_host }}", serverem na adrese 192.168.42.2 a metadata "linux bastion ansible". Spustil jsem instalaci příkazem `ansible-playbook -i inventory.ini install-zabbix-agent2.yml`, která úspěšně nainstalovala balíčky zabbix-agent2, zabbix-get a zabbix-sender, nakonfigurovala agenta a spustila službu. Po instalaci jsem ověřil funkčnost - `systemctl status zabbix-agent2` ukázal, že služba je aktivní a běží, `zabbix_agent2 -V` potvrdil verzi 7.0.24 a `ss -tlnp | grep 10050` ukázal, že agent naslouchá na IP 192.168.42.5 na portu 10050. Při pokusu o nahrání změn na GitHub jsem narazil na problém s autentizací, protože GitHub již nepodporuje přihlašování heslem. Vyřešil jsem to vytvořením Personal Access Tokenu na GitHubu s potřebnými právy a použil ho místo hesla. Nakonec jsem úspěšně nahrál všechny změny do repozitáře včetně screenshotů dokumentace. Výsledkem je plně funkční Zabbix Agent 2 nainstalovaný na hostiteli bastion, který je připraven k monitorování.
